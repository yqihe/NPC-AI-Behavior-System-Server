# 通用禁止红线

适用于所有项目。语言/框架专属禁令见对应文件（如 `go-pitfalls.md`）。

## 禁止静默降级

- **禁止**对无效配置值做静默降级（fallback to default）而不记录日志。配置错误必须在加载阶段报错或至少输出 WARN 日志
- **禁止**在运行时吞掉"不影响启动但影响行为"的错误。典型：条件 `op` 填错导致 `Evaluate()` 永远返回 false，系统正常运行但行为静默失效——比 crash 更难发现
- **禁止**在 lookup 失败时 silent return/continue。找不到注册类型、配置项、映射 key 时，必须记日志（至少 Debug 级别）

## 禁止安全隐患

- **禁止**将外部输入直接拼接为文件路径。用户发送的标识符可能包含 `../` 实现目录穿越，必须校验
- **禁止**配置错误延迟到运行时暴露。引用关系（如 A 引用 B 的 name）必须在加载阶段校验，不通过则启动失败
- **禁止**数据库查询中直接接受用户输入的操作符。查询条件必须用精确匹配，防止操作符注入
- **禁止**使用无超时的外部 IO 调用。所有网络请求、数据库操作必须带 `timeout`，防止 goroutine/线程永久挂起
- **禁止**信任前端/客户端校验。所有校验必须在后端重复执行，客户端校验仅作为 UX 优化
- **禁止**在 API 响应中暴露内部错误详情（堆栈、连接串、原始 error）。使用预定义错误描述，原始 error 写日志

## 禁止协作失序

- **禁止**收到协作方请求后不回复就开始自己的工作。先确认收到、评估可行性、给出计划
- **禁止**口头承诺不落文档。架构变更、接口约定必须书面记录
- **禁止**只修改本地参考文件（`configs/`）而不通过 API 写入数据库

## 禁止过度设计

- **禁止**引入没有使用场景的依赖或框架
- **禁止**添加没有对应实现的配置字段或环境变量。存在但无代码路径的配置 = 死配置
- **禁止**为只有一个调用点的逻辑创建抽象层
- **禁止**对不含敏感信息的内部通信做加密。零安全收益 + 纯增复杂度

## 禁止测试质量低下

- **禁止**测试中硬编码依赖外部数据的精确计数。如 `expected 4 transitions` 会因配置变更而假阳性失败。用 `> 0` 或检查具体属性
- **禁止**测试中忽略反序列化/解析错误。`json.Unmarshal(data, &v)` 的 error 必须检查，否则数据格式问题时测试静默通过
- **禁止**测试中存在未使用的变量或空断言（`if condition { /* 什么都不做 */ }`）。要么断言，要么删除
- **禁止**测试中存在死代码（未调用的 helper 函数、被注释的测试）
