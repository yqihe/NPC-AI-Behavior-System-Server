# 禁止事项（红线）

违反任何一条都意味着架构退化回 v1。

## 禁止硬编码业务逻辑

- **禁止**在 Go 代码中硬编码 FSM 状态定义或转换规则。FSM 状态枚举和转换条件必须从配置加载
- **禁止**在 Go 代码中硬编码事件类型与感知方式的映射（v1 的 `canPerceive()` switch-case）。感知规则必须随事件配置声明
- **禁止**在 Go 代码中硬编码 NPC 类型特定的参数（速度、感知距离等）。必须从配置加载
- **禁止**用 switch-case 做 NPC 类型分发（v1 的 `wireStates()`、`npcSpeed()`）。必须用注册表/工厂模式

## 禁止破坏层次

- **禁止** BT 反向驱动 FSM（v1 的 `KeyRequestedTransition`）。FSM 管"什么时候切状态"，BT 管"切了之后做什么"，方向单向
- **禁止** core/ 包 import runtime/ 或 gateway/。依赖方向必须单向向下
- **禁止** core/ 或 runtime/ 包 import experiment/（反向依赖）。experiment/ 可正向依赖 core/ 和 runtime/，通过 build tag 隔离，保证正常构建不含实验代码
- **禁止** gateway/ 承担非网络职责（v1 的 Gateway 里做了 AOI 计算、事件发布、Tick 调整）

## 禁止弱类型通信

- **禁止** Blackboard 使用裸 `map[string]any`。必须通过泛型 `BBKey[T]` 访问，编译期确定类型
- **禁止** Blackboard Key 散落在各文件中。所有 Key 必须集中定义在一个 const 文件（`keys.go`）中，声明名称、类型、读者、写者
- **禁止** Key 拼错静默失败。配置加载时必须校验所有引用的 Key 是否在注册表中存在，不存在直接 panic
- **禁止** FSM 状态名使用魔法字符串。状态名必须有枚举或 const 约束

## 禁止实验污染核心

- **禁止**核心代码中出现任何实验相关的字段、方法或 import（v1 的 Instance 中有 `pureBTEng`、`SetMode()`，Manager 中有 `SpawnDummies()`）
- **禁止**为了实验需求修改核心数据结构。实验通过外部组合核心组件实现

## 禁止实验作弊

来源：experiment-layer 设计中三次迭代的教训。

### 公平性

- **禁止**故意削弱对照组实现来制造差距。每种模式必须是该架构约束下的**最佳努力实现**。写完后自问："如果我是该方案的拥护者，我会怎么实现？"能轻松改进 = 在作弊
- **禁止**在对照组的 Go 代码中暗含实验组的核心能力

### 立论

- **禁止**把工业标配当创新点来验证。决策中心是标配，不需要用实验"证明它有价值"。实验要证明的是**架构模式**（三层协作）的价值
- **禁止**设计只能证明单一组件价值的实验。创新是三层协作，实验必须覆盖每一层的不可替代性——缺 FSM 会怎样、缺 BT 会怎样、缺 DC 会怎样

### 方法

- **禁止**先写代码再想验证。必须先回答"要证明什么命题"和"什么场景能区分 A 和 B"，再写代码
- **禁止**对"符合预期的数据"直接采信。先攻击它——改参数、极端输入、换场景
- **禁止**对"所有模式都 100%"的场景作为差异论据。等价场景只能说明"在此类场景下架构无差异"，不能支撑创新点
- **禁止**只用玩具规模验证架构优越性。如果论点是"A 比 B 扩展性好"，必须在不同规模（10/50/100/200）下测试并找到交叉点。5 个状态的场景无法论证规模优势
- **禁止**接受"全零"或"全相同"的指标数据。如果某个指标所有模式结果一致（如响应延迟全 0.0），说明指标设计有问题——要么量化方式错误，要么该指标对此实验无区分力，必须重新设计量化方式或替换指标

## 禁止安全隐患

来源：gateway-layer 代码审查发现的路径穿越漏洞。

- **禁止**将客户端输入直接拼接为文件路径。客户端发送的 `type_name`、`event_type` 等字段可能包含 `../` 实现目录穿越。所有用于文件路径的外部输入必须校验不含 `..`
- **禁止**配置错误延迟到运行时暴露。BT 树引用的 BB Key、FSM 引用的状态名等必须在加载/构建阶段校验，不通过则启动失败。禁止用运行时 panic 替代构建期校验

## 禁止过度设计

- **禁止**引入脚本引擎（govaluate、expr-lang 等）做条件求值。用简单规则匹配器（`bb_key + op + value/ref_key`，AND/OR 组合）
- **禁止**在没有使用场景时提前接入 Redis。v1 的 redis.go 是死代码
- **禁止**添加没有对应实现的配置字段或环境变量。`mongo_uri` 字段存在了多个版本但代码永远走 JSONSource 就是死配置。添加配置项时必须同步实现使用它的代码路径
- **禁止**为只有一个调用点的逻辑创建抽象层
- **禁止**对游戏模拟数据做传输加密或字段加密。WS 传输的全部是 NPC 坐标、行为状态、事件参数等游戏世界状态，无用户隐私、无认证凭证、无敏感信息，客户端必须明文获取才能渲染。加密 = 零安全收益 + 纯增复杂度
