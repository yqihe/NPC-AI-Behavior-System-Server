# 三个扩展轴与验证路径

## 扩展轴

所有设计必须服务于此：

1. **加事件源** → 所有已有 NPC 类型自动响应
2. **加 NPC 类型** → 自动处理所有已有事件
3. **NPC 间交互** → 决策中心统筹协调

## 验证路径

按以下路径逐步验证可扩展性，每一步预期结果都是"只改配置+加测试"：

1. 平民 × 3 事件源 → 基线
2. 平民 × 4 事件源 → 验证加事件源
3. (平民+警察) × 4 事件源 → 验证加 NPC 类型
4. (平民+警察) × 5 事件源 → 验证多类型自动响应新事件
5. 警察与平民交互 → 验证 NPC 间交互

如果某一步发现需要改核心 Go 代码，说明底层设计有缺陷，必须先修底层再继续。
